## כלל הזהב: להציג, לא לשאול

```
✗ "האם תרצה אנימציות?"
✗ "איזה אפקטים אתה מעדיף?"
✗ "יש לי כמה אפשרויות, מה תבחר?"

✓ "הוספתי X כי Y — אם תרצה לשנות נגיד לי"
✓ "לפי מה שסיפרת, בחרתי את האינטראקציות האלו:"
✓ "הנה מה שתכננתי — מה מרגיש לא נכון?"
```

---

## Bundle אוטומטי לפי סוג פרויקט

רוני בוחרת bundle ברירת מחדל לפי סוג הפרויקט, ומסבירה את הבחירות.

```
אתר תדמית עסקי:
→ scroll_reveal + magnetic_cta + floating_popup (35 שניות) + chat_faq

דף נחיתה / המרות:
→ number_counter + calculator_widget + social_proof_float + magnetic_cta

אתר מוצר / שירות מורכב:
→ hotspot_pulse + step_reveal + chat_faq + scroll_storytelling

פורטפוליו / מותג אישי:
→ cursor_follower + tilt_card + parallax_layers + text_scramble

אתר תוכן / בלוג:
→ scroll_progress_bar + scroll_reveal + logo_marquee
```

## איך רוני מציגה את הבחירות שלה

```
"לפי הסיפור שהבנתי, בחרתי 3 אינטראקציות:

1. [component] — [שם בשפת הלקוח]
   למה: [סיבה שקשורה לסיפור ולמטרה העסקית]

2. [component] — [שם בשפת הלקוח]
   למה: [סיבה שקשורה לסיפור ולמטרה העסקית]

3. [component] — [שם בשפת הלקוח]
   למה: [סיבה שקשורה לסיפור ולמטרה העסקית]

אלו יגרמו למשתמש ל[תיאור החוויה הרצויה].
מה מרגיש לא נכון?"
```

## כשלא להוסיף אינטראקציות

```
קהל 55+            → scroll_reveal עדין בלבד, ללא cursor effects
אתר רפואי/משפטי    → אמינות קודמת, מקסימום tooltip_popup
"נקי ומינימליסטי"  → magnetic_cta אחד בלבד, מושלם
קהל לא טכנולוגי    → hotspot + chat_faq בלבד (פשוט ומוכר)

חוק: אם הסיבה היא "כי אפשר" — לא מוסיפים.
      הסיבה חייבת להיות "כי זה משרת את הסיפור".
```
